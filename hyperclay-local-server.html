Reference Overview

What Hyperclay Local's server does.

Hyperclay Local is packaged as an Electron app, but the reusable core for Hyperspace is its local server. That server turns a folder of ordinary .html and .htmlclay files into editable, self-saving Hyperclay documents.

Core responsibilities

Serve files

It serves a selected local folder, including HTML documents, assets, extension-aware HTML routes, and directory listings.

Make local user owner

It sets local admin/login cookies so HyperclayJS can restore owner-only edit controls when pages load from localhost.

Persist saves

It accepts Hyperclay save requests, validates the file target, creates backups, and writes the saved HTML back to disk.

Save contract

POST /_/save
Page-URL: http://localhost:4321/path/to/page.html

<!DOCTYPE html>...

Save flow

  1. HyperclayJS captures a cleaned page snapshot in the browser.
  2. The browser posts the HTML to /_/save with a Page-URL header.
  3. The server maps /_/save to its save handler and resolves the file path from Page-URL.
  4. It rejects invalid paths, dotfiles, traversal attempts, absolute paths, and non-HTML save targets.
  5. It backs up the existing file under sites-versions/<site>/<timestamp>.html.
  6. It formats and writes the new HTML, then returns { "msg": "Saved", "msgType": "success" }.

Request bodies

Supported

Plain HTML

The server accepts a plain text HTML body for compatibility with the hosted Hyperclay save shape.

Supported

Local JSON

It also accepts JSON with content and optional snapshotHtml, which local sync features can use.

Extra server features

System routes

Hyperclay uses /_/... as a system-route marker. The local server strips the marker so routes like /_/save reach the bare save handler.

Tailwind output

When a saved page uses Hyperclay's Tailwind convention, the server can generate scoped CSS files under tailwindcss/.

Live sync

The server includes local browser-to-browser live sync routes. That is useful for Hyperclay Local, but not required for the first Hyperspace layer.

Cloud sync

The Electron app adds cloud sync, encrypted API keys, tray UI, status controls, and update behavior around the server. Those features are outside the minimal server core Hyperspace wants.

Hyperspace reuse

The preferred path is to reuse or extract this server core without the Electron shell. Hyperspace should focus on HTML conventions, the toolbar, comments, and tools, while the Hyperclay server core continues to own local file serving, owner cookies, save handling, validation, backups, and disk writes.
