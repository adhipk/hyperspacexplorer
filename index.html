Project Documentation

Single script editing for ordinary HTML pages.

HyperEdit is a small browser script that can be dropped into a page to turn the document itself into the editing surface. The page stays regular HTML; the script only provides an edit switch, highlight tools, and a save handoff for the server.

Design intent

The UI should feel like the minimalist highlighter sketch in the design document: a quiet floating rail, three simple colors, obvious edit on/off state, and no heavy application frame around the content.

What phase 1 does

Native editing

Edit mode applies native contenteditable to the configured root, usually main, so the document is the editor.

Small tools

The toolbar supports edit mode, highlights, color sampling, and manual save without requiring page authors to build UI.

HTML handoff

Saving serializes the current document and removes temporary editor state before sending it to a backend hook.

Consumer include

<script
  src="/hyperedit.js"
  data-root="main"
  data-save-url="/__hyperedit/save"></script>

Open question

A browser script cannot write the HTML file back to disk by itself. It needs either local draft storage or a serving backend that accepts the serialized HTML and persists the file.
