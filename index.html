Hyperspace

Review Loop Runtime Host

HTML Review Runtime

Review any HTML file without changing its source.

Hyperspace adds an external review layer to ordinary HTML: a toolbar, comments, structured list controls, quiet saves, and cleanup on persistence. Page styling stays with the HTML document; the review UI lives in hyperspace.js and hyperspace.css.

Review Loop

Serve a project folder, open any .html or .htmlclay file, then use the edit toggle, comment tool, and save button without embedding application chrome in the source document.

  1. Click the comment tool and place a note near the relevant content.
  2. Turn edit mode on only when text or list content needs direct changes.
  3. Save writes clean HTML back to disk with runtime UI removed.

Runtime Boundary

Ready Surface Boundary
External toolbar runtime. The toolbar, tool buttons, icons, list controls, and save toast styling are supplied by external runtime assets.
Clean artifact source. Saved HTML keeps document content, user comments, and durable transform state, but strips runtime-only scripts, links, toolbar nodes, and transient edit attributes.
Ordinary HTML compatibility. The local host can inject Hyperspace into any served HTML page, including pages that were not authored in this repo.

Host Contract

External assets

HTML responses receive a removable stylesheet link, HyperclayJS script, and Hyperspace script. Those tags are marked as runtime elements and are removed before the file is persisted.

Save cleanup

The save endpoint accepts Hyperclay-compatible page snapshots, validates the target path, creates a backup, strips runtime UI, and writes the resulting HTML atomically.
