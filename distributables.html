Distribution Notes

What Hyperspace should ship.

Hyperspace should distribute the smallest pieces that make agent-authored HTML artifacts reviewable, editable, and durable inside a consumer's own project.

Distributable files

  1. A lean browser script that can be included on any HTML page to add annotation and review tools.
  2. A backend server package that can run locally and persist durable HTML updates.
  3. Portable agent skills that can be installed in any repository, with instructions for using HTML artifacts as effective communication objects.

Documentation direction

Once the distributable shape is stable, consolidate existing engineering notes into a developer-docs area and create public documentation artifacts for distribution.

The documentation site should explain the tool while showcasing its power. It should be expressive and creative enough to demonstrate why editable HTML artifacts are useful, while still staying grounded in the actual review workflow.
