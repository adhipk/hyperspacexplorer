Decision Point

Durable annotations without corrupting page text.

HTML artifacts are live project documents. A coding agent owns the actual HTML file, and the file on disk is the source of truth. User comments, annotations, and targeted edits are first-class communication between editor and writer, so they must be durable, visible to the agent, and rendered back to the user as part of one cohesive page.

Chosen behavior

Comments are durable

When the user leaves a note, the agent must be able to read it from the artifact file or from durable data stored with that file.

Live editing is required

The user should be able to make small corrections and targeted edits without asking the agent for every change.

The page still feels whole

Users should see source content, comments, annotations, and pending edits rendered as one cohesive webpage.

Architecture decision

Whole-page edit mode is the wrong boundary. HyperEdit should let the user make targeted edits and annotations, store those changes as durable objects with the artifact, and let the agent reconcile them with the real HTML source.

Why separation matters

Need Meaning Risk if ignored
Agent-owned source The agent writes and revises the real HTML content. Notes or quick edits can become accidental source changes.
User-owned feedback The user can mark, organize, remove, and request changes quickly. The user needs the agent for every small correction.
Targeted edits The frontend records a focused change against a specific place. Whole-page editing can overwrite content the user did not mean to touch.
Rendered together The artifact reads as one annotated, editable webpage. Annotations feel like hidden metadata instead of communication.

Delta rule

User edits should be represented as targeted deltas or durable edit objects first, not immediate direct writes to the HTML source file. The agent can then apply, adjust, or reject those changes while keeping the source coherent.

Reset rule

Annotations attach to DOM elements. When an element is rewritten or replaced, its child annotations should be reset unless the system can clearly reattach them to the updated content.

Next implementation

  1. Keep saving comments to the HTML artifact so the agent can read them from disk.
  2. Move toward a stored annotation and edit layer inside the artifact, rather than note text sitting directly inside editable source content.
  3. Render the stored layer back onto the page so users see one cohesive document.
  4. Replace whole-page edit mode with targeted edit controls that cannot overwrite unrelated source text.
