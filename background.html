Background

The browser can be the frontend for any project.

Agents are already good at making rich HTML artifacts. Hyperspace turns those artifacts from static outputs into editable project surfaces: open the file, review it in place, save it back to disk, and let the agent continue from the source.

The Gap

Copying an HTML artifact back into Markdown collapses the loop.

The HTML-artifact pattern is strong because HTML can carry visual structure, interaction, spatial context, tables, diagrams, and domain-specific controls. But when the next action is copy, paste, or export-to-prompt, the browser is still just a preview.

Hyperspace makes the browser the place where the work happens. The artifact stays in the project. The user edits and marks it up directly. The agent reads the saved result from disk.

Working Model

Same HTML artifact. Better feedback path.

Hyperspace does not try to replace the agent, the code editor, or Hyperclay's save primitives. It adds the smallest useful review layer on top of ordinary HTML.

Current artifact loop

  1. 1Agent generates a rich HTML artifact.
  2. 2User opens it and reacts visually.
  3. 3User translates feedback into chat, Markdown, or copied text.
  4. 4Agent has to infer what changed.

Hyperspace loop

  1. 1Agent creates ordinary HTML in the project.
  2. 2Browser injects the external review runtime.
  3. 3User edits, comments, reorders, or marks up in place.
  4. 4Local save writes clean HTML back to disk for the agent.

Product Thesis

Keep the artifact readable. Move the tools outside it.

Ordinary source

The saved file remains readable HTML with durable comments and user edits, not an app bundle or opaque editor payload.

External controls

Toolbar UI, edit affordances, list controls, and toasts live in hyperspace.js and hyperspace.css.

Agent-native handoff

Skills tell coding agents how to author artifacts, interpret durable comments, and reconcile reviewed HTML back into work.

Showcase Surface

Do not copy the templates. Prove the loop.

The examples that inspired this project are useful references, but Hyperspace should showcase cases where in-place edits and durable review state matter.

Decision workbench

Compare proposals, mark the chosen path, and leave unresolved questions beside the relevant tradeoff.

Code comparison room

Review competing implementations with diffs, module maps, severity notes, and accepted edits.

Design critique wall

Show component states and interaction variants, then annotate exact moments or copy that need revision.

Durable config editor

Edit feature flags, prompts, routes, or JSON-like settings with constraints and save the reviewed result.

Project explainer

Turn a repository, incident, or subsystem into a readable map that the user can correct in place.

Triage board

Bucket tasks, bugs, research notes, or feedback into a live board where final state persists as HTML.
